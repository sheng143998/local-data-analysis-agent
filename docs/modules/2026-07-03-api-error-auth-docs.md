# 接口错误码与权限边界文档模块完成说明

模块：接口错误码与权限边界文档

当前状态：中文错误码与权限边界文档已补齐，README、`docs/api.md` 和 `docs/api_frontend_mapping.md` 入口已更新，验证已通过，准备随本次提交推送完成。

业务逻辑：本模块不修改任何接口实现、前端错误展示、鉴权逻辑或数据库结构，只记录当前 API 的真实错误响应、前端错误处理方式、普通业务接口和开发者调试接口的权限边界。文档明确当前没有登录鉴权层，`/api/runs` 和 `/api/memories` 不进入普通用户主导航，上线或多人使用前需要补权限控制。

关键代码：本模块只涉及文档：

- `docs/api_error_auth.md`：新增接口错误码、权限边界、前端错误处理现状和上线前检查清单。
- `README.md`：增加错误码与权限边界文档入口。
- `docs/api.md`：在文档开头互链错误码与权限边界文档。
- `docs/api_frontend_mapping.md`：在文档开头互链错误码与权限边界文档。
- `docs/plans/2026-07-03-api-error-auth-docs.md`：记录本轮文档任务计划和边界。

数据契约：没有新增或修改 API 字段。文档记录当前错误响应格式主要为 FastAPI 默认 `{"detail": "错误说明"}`，并说明 `404`、`422`、`500`、前端 `response.ok` 通用错误以及未来建议的 `401`、`403`、`409`、`503` 使用边界。

验证：已通过 `npm run backend:test`，73 个测试通过；已通过 `npm run frontend:build`；已通过 `npm run test:e2e`。

风险/后续：当前只是文档，不实现鉴权。后续如增加登录、角色、API token、统一前端 client 或更细错误码，需要同步更新本文档、README 和接口文档。
