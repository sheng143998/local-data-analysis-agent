# 模块：本地管理员账号创建

## 已完成行为

本地 PostgreSQL 已创建开发调试管理员账号。登录接口使用标准邮箱格式作为账号标识，因此最终登录邮箱为 `admin@example.com`，显示名为 `admin`，角色为 `admin`，账号状态为 `active`。账号密码由用户指定，仅在本次本地操作中用于生成密码哈希，未写入仓库文件或本文档。

## 关键决策

- 初次使用的 `admin@localhost` 被登录请求的邮箱校验拒绝，因此改用可由前端和 `LoginRequest` 接受的 `admin@example.com`。
- 账号创建或重置时会撤销该用户的既有会话，避免密码变化后遗留会话继续有效。
- 这只是本机开发账号，不构成生产账号初始化方案。

## API 与数据影响

- 仅修改本地数据库 `app_users` 与该账号关联的 `auth_sessions`。
- 未修改 API 路由、Pydantic 契约、数据库迁移、前端或环境配置。

## 验证

- 查询 `app_users`：账号邮箱、显示名、角色和状态分别为 `admin@example.com`、`admin`、`admin`、`active`。
- 调用 `POST http://127.0.0.1:8000/api/auth/login` 验证成功，响应用户邮箱为 `admin@example.com`、角色为 `admin`。

## 剩余风险与后续

- 用户指定的本机开发密码强度较弱，禁止复用于共享或生产环境。
- 后续如需要可增加受控的管理员初始化命令，避免直接执行本地运维脚本。
