# 本地管理员账号创建计划

## Goal

在本地数据库创建或重置一个用于开发调试的管理员账号，显示名为 `admin`，登录标识为 `admin@example.com`，角色为 `admin`。

## Scope

- 通过现有认证数据表创建或更新本地账号。
- 使用现有密码哈希能力保存用户指定密码，不保存明文密码到仓库文件。
- 验证账号可通过既有登录接口认证，并确认角色为 `admin`。

## Out of scope

- 不修改认证 API、数据库迁移、前端登录页面或角色模型。
- 不提交 `.env`、会话令牌、密码哈希或评测工件。
- 不创建生产环境账号。

## Implementation steps

- [x] 检查本地 `app_users` 中是否已有本地 `admin` 账号。
- [x] 创建账号并设置显示名、角色和密码哈希；保留会话撤销逻辑。
- [x] 调用登录接口验证邮箱、密码和管理员角色。
- [x] 记录操作结果与验证证据。

## Validation plan

- 查询账号的邮箱、显示名、角色和状态，不输出密码哈希。
- 使用 `POST /api/auth/login` 验证 `admin@example.com` 能登录，并检查响应角色为 `admin`。

## Risks

- `123456` 是弱密码，仅适用于本机开发环境；不得用于共享环境或生产环境。
- 重置既有账号会使该账号现有会话失效。
