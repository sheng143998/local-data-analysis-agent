# 明确数据问题的错误澄清防护

## Goal

避免意图模型将已有明确业务对象、聚合或排行条件的数据问题误判为“缺少分析目标”，同时保留真正模糊问题的澄清行为。

## Scope

- 当模型未给出任何业务候选却要求澄清时，基于原问题中的明确查询形态作受控回退。
- 回退仅保留原问题为自然语言业务候选，不生成或执行 SQL，不引入固定 SQL。
- 补 parser/澄清回归并重跑认证真值集，比较前后报告。

## Out of scope

- 不取消概览型、闲聊型或缺失业务对象问题的澄清。
- 不扩大预置词表作为唯一理解来源，不更改 Semantic Contract、Guard 或 Executor。

## Implementation steps

- [x] 为明确数据查询定义最小可解释的语义形态检测。
- [x] 在模型空候选错误澄清时回退为自然语言候选，并增加测试。
- [x] 运行意图/会话/SQL 选择 focused tests。
- [x] 重跑已认证 50-case 真值集并比较报告。
- [x] 记录、提交并推送。

## Validation plan

- 意图解析和会话续接 focused pytest。
- 认证 `database_ground_truth_questions.jsonl` 全量评测输出到独立报告。
- `git diff --check`。

## Risks

- 规则过宽会压制必要澄清；因此只针对“模型候选为空 + 模型主动澄清 + 原问题含明确查询操作与业务对象”三者同时成立的情况。
