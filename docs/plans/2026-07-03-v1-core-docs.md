# V1 核心文档补齐计划

当前正在做：核心文档已补齐且验证通过，准备提交并推送。

## Goal

让项目不仅有模块完成记录，也有稳定的 V1 核心说明文档，便于后续开发、评估和验收。

## Scope

- 包含：
  - `docs/architecture.md`
  - `docs/data_model.md`
  - `docs/agent_workflow.md`
  - `docs/sql_guard.md`
  - `docs/sql_memory.md`
  - `docs/evaluation.md`
  - README 文档入口。
  - handoff 和模块完成说明。
- 不包含：
  - 改动业务逻辑。
  - 生成完整产品手册。
  - 前端开发者页面。

## Module Boundary

- 上游输入：`executable-plan-draft.md`、当前 handoff、当前代码和评估报告。
- 输出：可被 README 链接的核心 Markdown 文档。

## Business Logic

- 新开发者先读 README，再按主题进入架构、数据模型、Agent 工作流、安全、记忆和评估文档。
- 文档明确当前已实现能力和仍是 V1 风险的部分，避免把确定性模板、默认关闭的模型路径或链路评估误写成完全完成。

## Data Contract

- 文档不改变 API 或数据库契约。
- 文档引用当前真实文件路径、API 路由、表名、脚本和命令。
