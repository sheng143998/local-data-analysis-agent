# 十个多表查询逐条验证

## Goal

生成并逐条验证十个覆盖核心业务关系的多表数据问题；每条均通过完整受控 SQL 链路，发现确定性问题时修复并从当前条重新验证，最终交付逐条报告。

## Scope

- 选择订单-支付、订单商品-商品、订单商品-成本、退款-订单、用户-订单、流量-订单/支付等十个多表问题。
- 逐条运行真实分析链路，记录意图、合同、候选 SQL、Inspector、Guard、EXPLAIN、只读执行、结果行数和耗时。
- 仅对可复现的解析、合同、Planner、Inspector、Guard 或执行问题修改代码；每次修改后运行 focused regression 并重测当前条。
- 输出可追溯的逐条报告；不把本地评测 JSON 工件纳入提交。

## Out of scope

- 不运行并发或批量评测，不用固定 SQL 模板绕过模型、合同、Guard 或执行器。
- 不把云端模型超时、空响应或非结构化输出伪装为业务口径通过。

## Implementation steps

- [x] 定义十个多表问题及其预期关系、结果形态和安全验证标准。
- [x] 按编号顺序生成、验证和记录每一条查询；失败时先诊断并修复后重测当前条。
- [x] 汇总逐条结果、失败分类、修复项与未解决的外部服务风险。
- [x] 运行改动对应的 focused tests、最终差异检查和前端构建。
- [x] 更新模块记录与 handoff，提交并推送。

## Validation plan

- 每条查询依次检查 Query Plan/Inspector、Guard、EXPLAIN 和只读 Executor，保留最小候选 SQL 摘要和结果摘要。
- 代码修改后运行相关 focused pytest；最终运行 `npm.cmd run frontend:build` 和 `git diff --check`。

## Risks

- 云端 SQL 模型可能在单条生成或修复时超时，需记录为外部失败，不能通过批量重试或固定 SQL 覆盖。
- 多表指标可能存在合同缺口或口径冲突；修复必须以版本化合同和确定性校验表达，避免只针对单个问题硬编码。
