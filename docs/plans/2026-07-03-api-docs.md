# V1 接口文档补齐计划

当前正在做：中文接口文档已补齐且验证通过，准备提交并推送。

## Goal

只补齐 V1 当前已实现接口的中文文档，让后续前端、后端和测试人员能快速确认接口用途、请求参数、响应结构、错误边界和普通用户/开发者接口边界。

## Scope

- 包含：
  - `docs/api.md` 中文接口文档。
  - README 文档入口和 API 说明入口。
  - handoff 和模块完成说明。
- 不包含：
  - 修改后端接口实现。
  - 新增接口。
  - 新增 embedding、模型、SQL 生成、模板或数据库能力。
  - 修改前端页面。

## Module Boundary

- 上游输入：
  - `backend/app/api/*.py`
  - `backend/app/schemas/*.py`
  - `backend/app/services/*.py`
  - `README.md`
  - `docs/handoff/current.md`
- 输出：
  - 面向项目协作的中文接口文档。

## Business Logic

- 普通业务接口包括健康检查、聊天式数据问答和指标口径 CRUD。
- 开发者调试接口包括运行记录和 SQL Memory 查询，不放入普通用户主导航。
- 文档必须说明 `/api/analyze` 返回的是可信分析结果，内部模型、数据库连接状态、SQL Memory 分数和评估报告不进入普通用户界面。

## Data Contract

文档覆盖以下当前接口：

- `GET /api/health`
- `POST /api/analyze`
- `GET /api/metrics`
- `GET /api/metrics/{metric_id}`
- `POST /api/metrics`
- `PUT /api/metrics/{metric_id}`
- `DELETE /api/metrics/{metric_id}`
- `GET /api/runs`
- `GET /api/runs/{run_id}`
- `GET /api/memories`
- `GET /api/memories/{memory_id}`
