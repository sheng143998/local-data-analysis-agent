# README 简历展示优化

## Goal

重写项目 README，使其适合作为简历项目链接的公开介绍，首屏突出业务价值、技术架构、量化效果和可复现入口，同时保持技术细节与内部文档可追溯。

## Scope

- 更新根目录 `README.md` 的结构、表述和量化结果。
- 核对启动命令、目录、核心文档和评测报告路径。
- 更新项目交接与模块完成记录。

## Out of scope

- 不修改应用代码、数据库迁移、API 合同或评测脚本。
- 不新增固定 SQL、模型 fallback 或安全边界例外。

## Implementation steps

- [x] 读取当前项目结构、最新评测结果和现有 README。
- [x] 重写 README 首屏、架构、能力、指标和运行说明。
- [x] 校验链接、命令和量化结果与仓库现状一致。
- [x] 更新模块记录和 handoff。

## Validation plan

- 使用 UTF-8 检查 README 文本编码。
- 校验 README 引用的文件、报告和 npm 命令均存在。
- 运行文档相关的静态路径检查，不运行与文档无关的全量模型评测。

## Risks

- 云端模型评测具有非确定性，README 只引用已落盘报告中的测量结果并标注评测集规模。
- 工作区有大量未提交业务改动，编辑时只触碰 README、计划、模块记录和 handoff。
