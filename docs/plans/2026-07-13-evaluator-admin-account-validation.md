# Evaluator Admin Account Validation

## Goal

在不读取、输出或修改密码与密钥的前提下，确认本机数据库中可用于 authenticated evaluation 的管理员账号条件，并验证评测脚本所需的最小配置路径。

## Scope

- 检查评测数据集、登录流程、`/api/runs` 的管理员权限和用户状态。
- 仅查询用户的邮箱、角色、状态等非敏感字段。
- 给出由主线配置未跟踪 `EVAL_AUTH_*` 后运行 50 case 基线的最小安全步骤。

## Out of scope

- 不读取或修改 `backend/.env`、任何密码、哈希、连接串或模型密钥。
- 不创建、重置、提权或删除用户，不提交文件。
- 不修改评测、鉴权或数据库实现。

## Implementation steps

- [x] 阅读项目 Skill、当前 handoff 和总执行计划。
- [x] 复核评测登录与 run trace 的权限需求。
- [x] 以最小只读查询确认可用管理员账号状态。
- [x] 验证评测脚本的配置阻断行为，并记录主线配置步骤和阻塞项。

## Validation plan

- 静态核对 `run_eval.py`、认证路由、权限依赖和 repository 查询。
- 使用数据库只读查询确认账号邮箱、角色与状态，不输出密码哈希。
- 仅在不写入配置的前提下验证缺失 `EVAL_AUTH_*` 时的明确阻断。

## Risks

- 账号存在不代表主线持有正确密码；登录失败必须归类为评测环境配置问题。
- 评测会实际调用模型和数据库，可能耗时较长；本子任务不直接运行完整 50 case。
